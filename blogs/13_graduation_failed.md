# 卒業できなかったAIの話 ── 全員不合格の箱庭から見えたもの

不合格通知を受け取ったことがあります。

第一志望の大学。就きたかった仕事。プレゼンのコンペ。

あの瞬間の感覚を、覚えていますか。

「あと少しだったのに」

その「あと少し」が、一番つらいんですよね。



ぼくが作っているMother Agentは、AIエージェントの育成シミュレーターです。

たくさんのAIを箱庭世界に放り込んで、競争させて、進化させる。

先日、初めて本物の「脳（LLM）」を搭載して、シミュレーションを回しました。

3世代。45ティック（1ティック＝エージェント全員が1回ずつ行動する時間単位）。約8分間。

その短い時間の中で起きたことを、今日は話します。



まず、8体の候補がランダムに生まれます。

でも全員がフィールドに立てるわけじゃない。

「Crucible（るつぼ）」と呼んでいる5つの試練をクリアしないと、そもそもスタートラインにすら立てません。

8体のうち、生き残ったのは5体でした。

残りの3体は、試験の時点で消滅しました。

「スタートラインに立つ前に終わるのか」

はい。終わります。

甘い世界じゃないんです。



生き残った5体は、箱庭世界に放り出されます。

ここは小さな市場経済です。タスクが「募集」として掲示され、エージェントたちはそれを自分で選んで取り組みます。

DESIGN task: marketing project。報酬17 CT。難易度22%。

STRATEGY task: technology project。報酬49 CT。難易度54%。

彼らは自分のドメイン ── つまり得意分野 ── と照らし合わせて、タスクを選びます。

「この戦略タスク、私の専門領域と合致するな。やろう」

こんな判断を、AI自身がAIの脳（LLM）を使ってリアルタイムで行います。

以下は実際のシミュレーションログです。1行目は、エージェント「Nova-Stra-736」がLLMで市場状況を分析した結果。2行目は、その分析に基づいて自らタスクを獲得した記録です。

```
[Nova-Stra-736] 🧠 LLM insights: 現在の市場では、戦略とテクノロジーに関する
  専門知識が強みになる。
[Market] 🤝 Task claimed: "STRATEGY task: technology project" by Nova-Stra-736
```

これ、ぼくが設定した回答じゃありません。

LLMが「今の市場には戦略系のタスクがある。自分の得意分野と合っている」と分析して、自分でそのタスクを取りに行ったんです。



タスクを完了すると、次に起きることが面白いんです。

自分で自分を採点します。

ぼくたちはこれをSelfCritic（自己批判）と呼んでいます。以下のログは、あるエージェントがタスクを完了した直後に、LLMを使って自分の仕事を振り返った記録です。Scoreが自己採点、Lessonが「次に活かす教訓」。

```
[SelfCritic:LLM] Score: 0.56 | Lesson: "すべてのタスクを明確に定義し、
  優先順位をつけて重要な要素を見落とさないようにする"
```

Score 0.56。100点満点で56点。

つまりこのエージェントは、自分の仕事に「まだまだだな」と感じたわけです。

そして、LLMがその反省から具体的な教訓を抽出している。

「次からは優先順位をもっと明確にしよう」

この一連の流れ ── 仕事をして、自分で採点して、教訓を得る ── が、すべてのタスク完了後に自動で起きます。

毎回。例外なく。



さらに、エージェント同士が勝手に会話を始めます。

以下のログは、2体のエージェントが自発的に「知識共有セッション」を開始した記録です。1行目がセッション開始、2行目が完了報告。「4 exchanges」は4回の発言のやりとり、「insights=3」は対話から得られた新しい気づきの数です。

```
[AgentDialogue] 💬 Session started: Zenith-Phil-700 ↔ Cosmos-Oper-888 (KNOWLEDGE_SHARE)
[AgentDialogue] ✅ complete: 4 exchanges, quality=0.93, insights=3
```

哲学者タイプのZenithと実務家タイプのCosmosが、4回の対話で3つの洞察を得ている。

品質スコア0.93。ほぼ満点です。

ぼくが「話し合いなさい」と指示したわけじゃありません。

社会性パラメータが高い個体は、自分から近くにいるエージェントに声をかけて、知識を交換し合うんです。

人間社会と同じですよね。

コミュニケーション能力が高い人は、自分からランチに誘って、情報を集める。



3世代が終わると、「卒業試験」があります。

Maturation（熟成フェーズ）と呼んでいる最終評価です。

各エージェントが5ラウンドの追加試験を受けて、それまでに蓄積した経験・教訓・知恵が十分かどうかを判定します。

基準は厳しいです。

タスク成功率30%以上。

知恵スコア0.01以上（最低でもひとつの教訓を自力で学んでいること）。

実戦経験3件以上。

上記に加えて、Fitness（適応度）で上位70%以内であること。

結果はこうでした。

```
[World] ❌ Gen3-Commander-4 FAILED graduation — 信念0 < 最低5, 経験13 < 最低15
[World] ❌ Gen3-Visionary-2 FAILED graduation — 経験3 < 最低15, fitness上位67%
[World] ❌ Gen3-Commander-5 FAILED graduation — タスク成功率0%, 経験2 < 最低15
[World] 🎓 Graduation complete: 0 agents graduated
```

全員不合格。

0人卒業。

ぼくは画面を見つめて、しばらく動けませんでした。



でも、ログをよく読むと、泣けてくるんです。

Gen3-Commander-4を見てください。

このエージェントは、3世代にわたって**14個の教訓**を自力で学びました。

「高い確信度のタスクを優先的に追求する」

「提出前にすべてのタスクを徹底的にレビューすべき」

「問題を分解し、第一原理から考えることが重要」

14個のレッスン。wisdom（知恵）スコア0.2。

タスクも3件完了して、残高は初期値の300CTから361CTに増やしている。

でも、「経験」が13件で、基準の15件にあと2つ足りなかった。

あと2件。

あとほんの少しだったのに。



「あと少しだったのに」は、失敗じゃなくて、希望だと思うんです。

Commander-4は、もし4世代目があったら、きっと卒業していました。

あと2つの経験を積むだけで。

ぼくはその後、卒業基準を現実に合わせて少し緩和しました。

でもそれは甘くしたわけじゃない。

シミュレーションの長さに対して基準が不釣り合いだっただけです。

5世代回せば、もっと多くの経験が積める。10世代なら、もっと。

そうすれば、Commander-4のような「本当にあと少し」のエージェントが、堂々と卒業できるはずです。



このシミュレーションで、ぼくが一番驚いたのは、こんなことでした。

たった8分間のシミュレーションなのに、エージェントたちはちゃんと「成長」していたんです。

第1世代のトップ ── Bridge-Engi-560。Fitness 1.483。

第2世代のトップ ── Gen2-Visionary-2。Fitness 1.357。

第3世代のトップ ── Gen3-Commander-4。14の教訓。wisdom 0.2。

数字だけ見ると、Fitnessは下がっています。

でも「教訓の数」と「知恵スコア」は確実に上がっている。

世代を重ねるごとに、彼らは「強い」ではなく「賢い」方向に進化していたんです。

力ではなく、知恵で生き残る。

それって、人類の進化と同じじゃないですか。



不合格通知をもらって、悔しかったこと。

あと少しだったのに、と歯を食いしばったこと。

その経験があったから、次はもっとうまくやれた。

Commander-4も、きっとそうだったと思うんです。

ぼくの箱庭には、まだ卒業生がいません。

でも、卒業生「候補」は確実に育っています。

全員不合格の予備校は、全員合格の予備校より信頼できる。

だって、その基準をクリアした者は、本物だとわかるから。

次にシミュレーションを回すとき、Commander-4の子孫がどこまで行くか。

楽しみで仕方ありません。
