# Vibe Codingが止まる時


いつものようにコードを書いてたのね。

AIとのペアプログラミング。俺の思考がそのままコードになり、機能として動き出すまでのタイムラグがゼロに近づく瞬間。これが一番気持ちいい。

「ここ、もっとこういう感じで」
「了解、修正しました」

この高速のラリーこそが、俺の言う「Vibe Coding」の醍醐味。でも、その最高のマリアージュを邪魔する奴がいる。

npm install だ。



俺たちの開発環境はポータブルなのね。「どこでも開発できる」を実現するために外付けSSDを使ってるんだけど、これには弱点がある。

npmが node_modules という名の数万個の小ファイルを書き込み始めた瞬間、SSDは悲鳴を上げ、Finderは虹色の円盤を回し始める。

AIとの会話は弾んでいるのに、PCが追いつかない。

まるで、高速道路を走っていたのに突然渋滞に巻き込まれたような感覚。この「待ち時間」が、せっかくのVibe（没入感）を完全に断ち切ってしまう。



で、俺は隣にいるAIパートナーに愚痴をこぼしたのね。

「npm install、遅すぎない？ SSDも熱いし」

すると彼は、こともなげに提案してきた。

「pnpm に変えましょう。ディスク容量も節約できるし、インストールは一瞬になりますよ」

え、そんな簡単に変えられるの？

そう思っている間に、彼はマイグレーションプランを提示してきた。これは単なる手順書ではなく、「どのファイルを削除し、どこを書き換えるか」という具体的な設計図。俺がそれを「いいね」と承認するだけで、彼はコマンドを実行し始めた。



もちろん、スムーズにいかないこともある。

「……あ、エラー出ましたね」

Gitにコミットしようとした瞬間、見たこともないログが流れた。

普通ならここでブラウザを開き、Stack Overflowを彷徨い、似たような症例がないか何時間も検索するところ。

でも、Vibe Codingでは違う。「これ何？」と聞くだけ。

「これはmacOSが外付けドライブに作成する隠しファイルが、Gitと競合しているようです。修正しますね」

次の瞬間には、修正コマンドが走っていた。俺がエラーログを読み解くよりも早く、彼は原因を特定し、解決策を実行していた。

このトラブルシューティングのスピード感。これこそが、Vibe Codingの本質かもしれない。



修正が終わり、再びインストールコマンドを実行した時、俺は耳を疑った。

一瞬で終わったのだ。SSDのアクセスランプもほとんど点滅しない。

「どうですか？」
「最高。これでまた、Vibeに乗れるね」

PCのファンは静まり返り、画面の中では再び高速なコード生成が始まった。



もしあなたが「開発環境が重い」「セットアップが面倒」と感じているなら、それはマシンのせいではなく、対話不足かもしれない。

困ったら、AIに聞いてみて。「もっと快適にコード書きたいんだけど」って。

きっと、想像以上の答えが返ってくるはずだから。
