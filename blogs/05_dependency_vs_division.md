# 依存じゃなくて、分業


「AIに頼りすぎじゃない？」

最近、そう言われることが増えた。AIなしでコード書けるの？ AIがなくなったらどうするの？ それって依存症じゃない？って。

俺はね、逆だと思うわけです。



まず「依存」って言葉の意味を考えてみる。

依存って、「なくなったら困る」ってことでしょ。たしかに、AIがなくなったら不便にはなる。でも、困るか？

電卓がなくなったら困る？ 手計算すればいい。遅くなるけど、できる。

エクセルがなくなったら困る？ 紙に書けばいい。面倒だけど、できる。

AIも同じ。なくなったら遅くなるけど、コードは書ける。だから「依存」じゃない。「効率化ツール」を使ってるだけ。



で、もっと大事なのは「何を任せているか」なのね。

俺がAIに任せてるのは、「俺がやりたくないこと」。

ゴミ掃除。整理整頓。同じコマンドの繰り返し。エラーログの解読。ドキュメントの検索。こういうの、人間がやる意味ある？

俺がやるべきなのは、「何を作りたいか」を決めること。どんな機能が必要か。どんな体験を提供したいか。どんな問題を解決したいか。

これはAIにはできない。だって俺の「欲しい」は俺にしかわからないから。



この役割分担がはっきりしてる限り、依存じゃなくて「分業」。

むしろ、全部自分でやろうとするのが傲慢なんじゃないかなって思う。

料理人が自分で畑を耕す必要ある？ 作家が自分で印刷機を回す必要ある？ 画家が自分で絵の具を作る必要ある？

できることと、やるべきことは違う。



昔、「真のエンジニアはIDEを使わない」みたいな風潮があった。vimとかemacsで全部やるのがイケてる、みたいな。

今思えば、あれって「苦労美学」だったのね。便利なツールを使わないことで、自分の「実力」を証明しようとしてた。

でもさ、それって本末転倒じゃない？

エンジニアの価値って、「どれだけ苦労したか」じゃなくて、「どれだけ良いものを作ったか」でしょ。

だったら、使えるツールは全部使えばいい。AIだろうがIDEだろうが、良いアウトプットが出ればそれでいい。



「でも、AIに頼ってると自分のスキルが伸びないんじゃない？」って思う人もいるかもしれない。

俺の経験では、逆。

AIに下のレイヤーを任せることで、俺は上のレイヤーを考える余裕ができる。

コマンドを覚える時間をAIに任せれば、俺は「なぜそのコマンドが必要か」を理解する時間ができる。エラーの検索をAIに任せれば、俺は「なぜそのエラーが起きたか」を考える時間ができる。

つまり、「作業」を減らして「思考」を増やせる。これってスキルの成長じゃない？



依存と分業の違いは、「主体性」だと思う。

依存は、判断を他者に委ねること。
分業は、判断を自分で持ちながら、作業を分担すること。

俺は判断を手放していない。何を作るか、どう作るか、最終的にOKを出すか。全部俺が決めてる。

だから依存じゃない。分業。



もし「AIに頼りすぎかも」って不安になったら。

自分の「判断」は自分でできてるか？ を確認してみて。

それができてれば、大丈夫。あなたは依存してるんじゃなくて、賢く分業してるだけ。
