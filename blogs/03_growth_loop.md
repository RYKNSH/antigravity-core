# 明日の俺は、今日より速い


昨日書いたコードを、今日見返して恥ずかしくなったことある？

俺はしょっちゅうある。「なんでこんな書き方したんだ」「ここ、もっとスマートにできたじゃん」って。でも最近、その「恥ずかしさ」の頻度が減ってきたのね。

理由は簡単。俺のAIパートナーが、「学んで、良くなる」仕組みを持っているから。



たとえば今日、俺がバグを直したとする。

普通ならそこで終わり。「直った、次いこう」ってなる。でも俺の開発環境は違う。直した後に、自動で「検証ループ」が回る。

まずテストが走る。リンターが走る。ビルドが通るか確認する。ここまでは普通。

違うのはその後。「このバグ、なぜ開発中に見つからなかったのか」を問いかけてくるのね。

テストで防げたか？ → 防げるとしたらどんなテストか？
設計で防げたか？ → どういう設計なら発生しなかったか？
プロセスで防げたか？ → チェックリストに追加すべき項目はあるか？

この振り返りが、ナレッジとして保存される。だから次に似たようなバグが発生しそうな時、先回りで「ここ、前に問題になったパターンに似てますね」って教えてくれる。



これ、俺は「知識の複利」って呼んでるのね。

銀行に預けたお金が利息で増えていくように、俺の「学び」も蓄積されて増えていく。今日の1時間のデバッグが、明日の3秒になる。今日の設計ミスが、来月の回避パターンになる。

問題は、普通この「複利」って働かないこと。

だって人間って忘れるから。3ヶ月前のバグの原因なんて覚えてない。半年前に「こうしよう」って決めたルールも、いつの間にか守られなくなってる。

でもAIは忘れない。ナレッジベースに保存された学びは、永久に参照可能。だから複利が効く。



もう一つ面白いのが、この成長ループが「自動」で回ること。

俺が毎回「振り返りをしろ」って命令してるわけじゃない。ワークフローの中に組み込まれているから、勝手に実行される。

朝「チェックイン」って言うと、環境が整う。一日の終わりに「チェックアウト」って言うと、今日の学びがまとめられる。俺は「おつかれ」って言うだけでいい。

この「自動化」がないと、振り返りって続かないのね。人間って弱いから、面倒なことはサボる。でも仕組みに組み込まれてれば、サボりようがない。



ここで「でも、それって機械任せで自分は成長しないんじゃない？」って思う人もいるかもしれない。

俺はね、逆だと思うわけです。

機械に任せることで、俺は「もっと上のこと」を考える余裕ができる。

振り返りの記録を取る時間をAIに任せれば、俺は「この学びをどう活かすか」を考えられる。パターンの整理をAIに任せれば、俺は「新しいパターンを発明すること」に集中できる。

つまり、AIが下のレイヤーを固めてくれるから、俺は上のレイヤーに登れる。これって、成長の加速装置だと思うのね。



明日の俺は、今日より速い。

それを確信できるって、最高に気持ちいいこと。

もし今、「毎日同じことの繰り返し」って感じてるなら。

学びを「蓄積」する仕組みを作ってみて。AIに手伝ってもらえば、意外と簡単にできるから。
