# 開発原則（Development Principles）

> **このドキュメントは、このプロジェクトの実装において絶対死守すべき現場のルールを定義する場所です。**
> Antigravityシステムの根幹をなす設計思想（First Principles, Persona Orchestration等）は `~/.antigravity/docs/WHITEPAPER.md` に一元化されているため、AIエージェントはそちらを上位教義として参照します。
> ここには「この特定プロジェクト固有の制約」のみを記載してください。

---

## 🚫 アンチ・バンドエイド原則（全プロジェクト共通チェックリスト）

AIおよび開発者は、変更をコミットする前に必ず以下を確認すること：

- [ ] この変更は根本原因を解決しているか？（First Principlesスキルを発動したか？）
- [ ] 同じ問題が他の場所で発生しないか？
- [ ] この変更なしで問題を解決する方法はないか？
- [ ] 6ヶ月後の自分がこのコードを理解できるか？（過度な複雑化をしていないか？）
- [ ] テストまたは検証プロセス（`/verify`, `/fbl` 等）を通じて機能が証明されているか？
