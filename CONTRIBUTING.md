# コントリビューションガイド

Antigravityへの貢献を歓迎します！

## 誰でも貢献できます

**エンジニアでなくても大丈夫です。**

以下のような貢献を歓迎します：
- 📝 ドキュメントの改善
- 🌍 翻訳
- 🐛 バグ報告
- 💡 機能提案
- ❓ 質問（FAQ充実に役立ちます）
- 📖 チュートリアル作成
- 🎨 デザイン改善

## 貢献の方法

### 1. ドキュメント改善

**最も簡単な貢献方法です。**

- タイポを見つけた → 修正してPull Request
- わかりにくい説明 → 改善案を提案
- 不足している情報 → 追加してPull Request

**やり方**:
1. GitHubで該当ファイルを開く
2. 「Edit」ボタンをクリック
3. 修正する
4. Pull Requestを作成

### 2. バグ報告

バグを見つけたら、[Issue](https://github.com/RYKNSH/antigravity-core/issues/new?template=bug_report.md)を作成してください。

**含めてほしい情報**:
- 何をしようとしたか
- 何が起きたか
- 期待していた動作
- エラーメッセージ（あれば）
- OS・環境情報

### 3. 機能提案

新機能のアイデアがあれば、[Issue](https://github.com/RYKNSH/antigravity-core/issues/new?template=feature_request.md)を作成してください。

**含めてほしい情報**:
- どんな問題を解決するか
- どう使うか（ユースケース）
- 代替案（あれば）

### 4. 質問

わからないことがあれば、[GitHub Discussions](https://github.com/RYKNSH/antigravity-core/discussions)で質問してください。

**「愚問」は存在しません。**

あなたの質問は、他の誰かも同じ疑問を持っている可能性があります。
質問することで、ドキュメント改善にも貢献できます。

### 5. コード貢献

コードを書ける方は、Pull Requestを歓迎します。

**手順**:
1. Forkする
2. ブランチを作成 (`git checkout -b feature/amazing-feature`)
3. 変更をコミット (`git commit -m 'Add amazing feature'`)
4. Pushする (`git push origin feature/amazing-feature`)
5. Pull Requestを作成

**コーディング規約**:
- シェルスクリプト: POSIX互換を優先
- ドキュメント: 非エンジニアにもわかる言葉で
- コメント: 「なぜ」を説明（「何を」は不要）

## Pull Requestのレビュー

Pull Requestは、メンテナーがレビューします。

**レビュー基準**:
- ✅ 非エンジニアに優しいか
- ✅ ドキュメントは更新されているか
- ✅ 既存機能を壊していないか
- ✅ ライセンス（AGPL v3）に準拠しているか

**フィードバックがあっても気にしないでください。**
改善のための建設的な議論を歓迎します。

## 行動規範

Antigravityは、誰もが安心して参加できるコミュニティを目指しています。

詳細は [CODE_OF_CONDUCT.md](CODE_OF_CONDUCT.md) をご覧ください。

- 🌈 多様性を歓迎
