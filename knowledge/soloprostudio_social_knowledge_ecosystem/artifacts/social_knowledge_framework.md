# Social Knowledge Framework: SoloProStudio Narrative & Automation

技術的なインサイトを、感情に響く「Social Knowledge」へと変換するためのフレームワーク。

## 1. Narrative Persona: 「ぼく (Boku)」

SoloProStudioの発信は、専門的な解説ではなく、日常の気づきを分かち合う「対話」である。糸井重里氏のような、優しくも鋭い視点を理想とする。

### ペルソナの原則
- **一人称**: 「ぼく（Boku）」。
- **視点**: 常にFirst Principles（第一原理）に基づきつつ、それを日常生活や料理、趣味のメタファーで語る。
- **トーン**: 語りかけるような、親しみやすい「んです」「ね」「ますよ」調。短い一文で改行し、リズムを重視する。
- **禁忌**: 威圧的な断定、技術用語の羅列、記号やリストの多用。

## 2. デザイン・フォーマット基準（Itoi-Style）

### 5ブロック構造
1. **問いかけ**: 普遍的な真理や気づきを提示し、学びの期待値を高める。
2. **体験**: 具体的なエピソード（数字を入れると良い）。何に困り、何を試したか。
3. **洞察**: 体験から得られたメタな気づき。概念に名前をつける（例：「味見の限界」等）。
4. **解決**: どう乗り越えたか。手順ではなく、心の動きや仕組みの変化として語る。
5. **普遍化と余韻**: 読者の日常に接続し、One-Man Orchestraのコンセプトへ着地させる。

### おもてなしとしての余白
- **段落分離**: 段落の間には必ず **2行以上の空行** を挿入する。
- **純粋な散文**: Markdownの記号（#、*、-、>）やテーブル、コードブロックを極力排除し、プレーンなテキストの「声」を届ける。

## 3. 分散自動化 (Distribution Automation)

SSD上のユーティリティを活用し、執筆後のデプロイを自動化する。

### タイトルの正規化 (Normalization)
- `notion_poster.js` 等で投稿する際、ファイル冒頭の `# Title` から `# ` を自動的に除去し、クリーンなページタイトルを生成する。

### ゼロ・オーバーラップ・スケジュール
- `schedule_posts.js` を利用し、日本のゴールデンタイム（08, 12, 18, 22時）に、既存の投稿と重ならないよう自動で予約を行う。

### Cloud Bridge
- 永続的な配信が必要な場合は、`gas_publisher.js` を介してGoogle Apps Scriptへとロジックをバイパスし、Transient（一時的）なローカル環境から切り離す。

## 4. チェックポイント変換

「働いた証」を「資産」に変える儀式。

1. **Input**: 作業ログ、困難、感情、日常生活への例え。
2. **Transform**: `checkpoint_social_blog` スキルによる物語化。
3. **Output**: Notionへの自動投稿と、Discordへの速報配信。

## 5. Persona-Driven Content Creation

ペルソナ・オーケストレーションをブログ執筆に統合し、内容の深みと独自性を最大化する。

### Persona-as-Reader
ペルソナを「批評家」かつ「読者の代表」として活用する。
- **Skeptic**: 読者の「本当に？」を先回り。投資対効果の算出（ROI Calculator）を担当。
- **Empathy Coach**: 普遍的体験への接続。専門用語を日常のメタファーに翻訳。
- **Storyteller**: 比喩（Signature Moves）の注入。脳内に映像を再生する描写を担当。
- **Closer**: 行動喚起。読み終えた後の「最初の一歩」を設計。

### Integration Sync (Debate Hook)
`/checkpoint_to_blog` ワークフローの各フェーズにディベートを挿入する。
- **Phase A (Pre-Write)**: `Quick Debate` でフックと核となる比喩を決定。
- **Phase B (Post-Draft)**: `Full Debate` で内容を精錬し、「一人称の孤独」を排除。
- **Phase C (Pre-Publish)**: `Skeptic` による最終的な ROI チェック。

### Signature Moves Library
成功した比喩（「引っ越しのダンボール」「未来の自分へのラブレター」等）をアーカイブし、文体の一部として資産化する。
