# REQUIREMENTS.md — 完全自律AI駆動型SSDネイティブ環境

> Compiled by Jensen 🧥 — 2026-02-10

## Vision Statement
**人間がいなくても勝手に動くSSD。** AIがコードを書き、エラーをゼロにし、UXを磨き、完成品だけを人間に届ける。

## Core Requirements

### R1: 完全自律実行 (Autonomous Execution)
- AIはタスクを受け取ったら**ゼロエラー到達まで**自律的にループする
- 「明らかなエラーが残ったまま報告が来る」を**根絶**する
- 検証 → エラー検出 → 修正 → 再検証 のサイクルを自動で回す
- lint, type check, build, test, UX review を**全て通過するまで終わらない**

### R2: Level 3 自律 + 承認ゲート
- **全自動**: コード変更、コミット、テスト、デプロイ
- **承認が必要な3領域のみ**:
  1. 親OSシステムへの変更（brew install, sudo等）
  2. 決済に関する操作
  3. アカウント権限の変更
- 承認はプッシュ通知で行い、応答待ちの間も他タスクを継続

### R3: SSDネイティブ
- 全ての状態、ナレッジ、設定がSSD上に完結
- どのMacに接続しても同一環境が再現される
- クラウド依存を最小化（API呼び出しのみ）

## Pain Points (現状の課題)
1. エラーチェックが不完全なまま報告される
2. UX体験レビューが自律的に行われない
3. デバッグが途中で止まり人間待ちになる
4. セッション間の文脈が失われる

## Success Criteria
- [ ] `npm run build` + `ruff check` + テスト全通過でないと報告しない
- [ ] UIの目視レビュー（ブラウザ自動起動→スクリーンショット→自己評価）が自律実行される
- [ ] 承認ゲート以外の全操作がSSD上で完結する
- [ ] 48時間放置しても環境が腐らない（自動クリーンアップ）
