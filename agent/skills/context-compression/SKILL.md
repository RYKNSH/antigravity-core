# Context Compression Skill

## 目的

長時間セッション・大規模プロジェクトでのコンテキスト劣化を防ぎ、AIエージェントの応答品質を維持する。

## Progressive Disclosure パターン

コンテキストを3層に分離し、必要な時に必要な分だけ読み込む:

### Layer 1: メタデータ（常に読み込み）
- ファイルのインデックス（名前+1行サマリー）
- 現在のタスク状態（.session_state）
- エラーカウント・進捗

### Layer 2: サマリー（タスク関連時に読み込み）
- 関連ファイルの構造概要（view_file_outline）
- 関連ナレッジのknowledge.md

### Layer 3: 詳細（作業対象時のみ読み込み）
- ソースコード全文
- テスト結果・ログ全文
- ナレッジのartifacts/

## 実行手順

### 1. コンテキスト棚卸し
```
現在のコンテキストに含まれる情報をリスト化:
- [ ] 必須（現在のタスクに直結）
- [ ] 参照（使う可能性がある）
- [ ] 不要（現在のタスクに無関係）
```

### 2. 不要コンテキストの解放
- 完了タスクの詳細コードは忘れてOK（サマリーのみ保持）
- 無関係なナレッジは参照しない
- ブラウザ録画・ログは最新のみ保持

### 3. サマリー化
長いコンテキストを構造化サマリーに変換:
```
# [ファイル名]
## 役割: [1文で説明]
## 主要関数: [関数名リスト]  
## 依存: [依存先]
## 注意点: [バグ・制約]
```

### 4. 状態の永続化
- `.session_state` にサマリーを追記
- Compaction発生時は `.session_state` から復元

## トリガー条件

- セッション開始から2時間経過
- 5ファイル以上を同時編集中
- レスポンスが遅くなった（体感）
- Compaction警告が表示された

## アンチパターン

- ❌ 全ファイルを一度にview_file
- ❌ knowledgeディレクトリを全スキャン
- ❌ 終了したタスクのコードを保持し続ける
- ❌ 巨大なログ出力をそのまま保持
