# High Conversion LP Architect

このスキルは、ユーザーが提供するターゲット情報と商品情報をもとに、Geminiを活用した「成約率特化型LP構成案」を作成するプロセスを自動化・ガイドします。

## Prerequisites

- ユーザーから以下の情報をヒアリングする必要があります（まだ提供されていない場合）。
    - **ターゲット**: どんな人に売りたいか（例: 「30代の多忙なビジネスマン」）
    - **商材（Product）**: 何を売るか（サービスの概要、特徴）

## Process

### 1. 顧客の深層心理特定 (Deep Search)

ターゲットの建前ではない「本音の悩み」を抽出します。

**Action:**
以下のプロンプトを実行し、結果をユーザーに提示します。

```markdown
「{{TARGET}}」が検索窓に打ち込む、秘匿性の高い悩みを30個出力してください。
建前ではなく、誰にも言えない本音の悩みを中心に抽出してください。
```

> [!TIP]
> 出力された悩みの中から、特にLPの軸になりそうなものをユーザーに選んでもらうか、Agentが推奨するものを提案してください。

### 2. PASTOR法構成案の作成 (Structure)

特定した悩みをもとに、購買心理に基づいた論理的な構成案を作成します。

**Action:**
以下のプロンプトを実行します。

```markdown
特定した悩み（選定されたもの）をもとに、以下のLP構成案を作成してください。

「問題提起から解決策、そして行動誘導まで、購買心理に基づいた論理的な順序（PASTOR法）で構成案を作成してください」
```

### 3. フック（キャッチコピー）生成 (Hooks)

ファーストビューの直帰率を下げるためのキャッチコピーを大量生成します。

**Action:**
以下のプロンプトを実行します。

```markdown
「{{TARGET}}」が1秒で指を止める、ベネフィットと数字を組み合わせたキャッチコピーを100案出してください。
```

> [!TIP]
> 100案すべてを表示すると長すぎる場合は、Agentが「特に効果的と思われる10選」をピックアップして提示し、残りはファイルに出力するなどの配慮を行ってください。

### 4. 客観的証拠とFAQ (Proof & Trust)

信頼性を担保するためのコンテンツを生成します。

**Action:**
以下のプロンプトを実行します。

```markdown
この商品（{{PRODUCT}}）が他社より優れている根拠をデータで5つ、購入者の不安を解消するFAQを10個作成してください。
```

### 5. デザイン指示書への変換 (Detailed Specs)

テキスト構成をデザイナー向けの実装指示書に変換します。

**Action:**
以下のプロンプトを実行します。

```markdown
この構成を、Webデザイナーが即座に形にできる詳細な配置指示書に書き換えてください。
各セクションの役割を定義し、以下を含めてください：
- 視覚的な強調ポイント（文字サイズ、色使いのヒント）
- レイアウトの指定（カラム数、配置）
- 使用すべき画像やアイコンのイメージ
```

### 6. ABテスト案の先行生成 (A/B Testing)

運用後の改善案を生成します。

**Action:**
以下のプロンプトを実行します。

```markdown
成約率が目標に達しない場合に備え、訴求の切り口を変えた別パターンのヘッドラインを5つ提案してください。
また、それぞれのヘッドラインがどのような心理効果を狙っているか解説してください。
```

## Output

最終的に、すべての生成結果をまとめたマークダウンファイル（例: `lp_plan_for_{product}.md`）を作成し、ユーザーに提供してください。
